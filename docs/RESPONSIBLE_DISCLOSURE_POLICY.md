# Responsible Disclosure Policy

Status: operational MVP policy. Legal review required before public launch.

## Default Position

wr3 practices passive analysis and private disclosure by default. Active
validation is allowed only for owned contracts, explicit written authorization,
or projects covered by a compatible Safe Harbor scope.

## Timeline

| Day | Action |
| --- | --- |
| 0 | Private contact through available channels |
| 7 | Escalate through trusted channel such as SEAL 911 if ignored |
| 14 | Prepare CVE/EUVD or equivalent coordination where applicable |
| 45 | Continue coordination and mitigation follow-up |
| 90 | Limited disclosure may be considered without working PoC |
| 180 | Full disclosure may be considered if unresolved and legally reviewed |

## Prohibited

- Publishing working exploit steps before the disclosure window.
- Mainnet exploitation.
- Public fraud/scam accusations based only on automated results.
- Sharing customer private findings with non-authorized parties.

## Case Tracking

Disclosure cases must include:

- Finding id.
- Project contact.
- Scope note.
- Contact log.
- Current status.
- Next deadline.
- Reviewer owner.

The API supports private reviewer-only disclosure case creation, contact log, and
status advancement.
